# 三连轮战车控制系统 - 项目功能说明

## 一、项目概述

本项目为基于 **STM32F407** 微控制器和 **FreeRTOS** 实时操作系统的 RoboMaster 战车控制系统。实现了遥控信号接收、4路舵机控制、BMI088 IMU 惯性测量，以及 USB 虚拟串口数据回传功能。

- **芯片**: STM32F407ZGT6
- **主频**: 168MHz (HSE PLL)
- **架构**: FreeRTOS (3个任务并发)
- **通信**: SBUS (UART3) + USB CDC (UART1) + SPI1 (BMI088)

---

## 二、硬件资源分配

| 外设 | 功能 | 配置 |
|------|------|------|
| UART3 | SBUS 遥控接收 | 100000bps, 9bit, Even parity, 2 stop bits |
| UART1 | USB 虚拟串口回传 | 115200bps, 8N1 |
| SPI1 | BMI088 IMU 通信 | Master, HIGH/2EDGE, 波特率不分频 |
| TIM1 | 舵机 PWM 输出 (PE9/11/13/14) | 50Hz, 周期 20000 |
| TIM8 | 备用 PWM 输出 | 50Hz, 周期 20000 |
| DMA1 Stream1 | UART3 接收 | 备用（本版本未启用） |
| GPIOH10/11/12 | 系统状态灯 (蓝/红/绿) | Output Push-Pull |
| PA4 | BMI088 加速度计片选 (CS1_ACCEL) | Output |
| PB0 | BMI088 陀螺仪片选 (CS1_GYRO) | Output |
| GPIOE9/11/13/14 | 舵机 PWM 输出 (C板接口) | TIM1 AF PP |

## 三、任务架构 (FreeRTOS)

### 3.1 任务列表

| 任务名 | 函数 | 优先级 | 周期 | 堆栈 | 功能 |
|--------|------|--------|------|------|------|
| LedTask | `StartLedTask` | osPriorityLow | 500ms | 128 | 系统心跳灯 |
| ControlTask | `StartControlTask` | osPriorityHigh | 10ms | 128 | 核心驱动控制 |
| ImuTask | `StartIMUTask` | osPriorityRealtime | 2ms | 256 | IMU采集回传 |

### 3.2 任务详解

#### LedTask (心跳灯)
每 500ms 翻转一次 PH10 (蓝色 LED)，用于指示系统正常运行。

#### ControlTask (核心控制 - 100Hz)
核心战车控制逻辑，包含两种模式：

**模式1：同步驾驶模式 (SwB < 1000)**
- 右手上下 (CH2) → 整体驱动速度
  - 1号大轮：按 50/63 比例换算
  - 3号/4号小轮：直出
  - 均叠加物理补偿值 TRIM_M1/M3/M4
- 左手上下 (CH3) → 转向角度 (500-2500，中心1500)

**模式2：纯右手调试模式 (SwB >= 1000)**
- 右手上下 (CH2) → 1号大轮独立控制
- 右手左右 (CH1) → 小轮测试速度
- 副开关 SwE (CH5) → 切换测试 3号轮 或 4号轮

**失控保护**: 遥控信号中断超过 50ms 时，所有舵机输出 TRIM 补偿值（防止车辆溜车）

#### ImuTask (IMU采集 - 500Hz)
- 启动时不断重试 BMI088 初始化，成功后等待 2 秒让 USB 枚举完成
- 循环读取 BMI088 角速度、加速度、温度数据
- 通过 USB CDC 发送格式化字符串 (需 USB 连接电脑后才发送)

## 四、模块说明

### 4.1 SBUS 遥控接收 (`sbus.c`)

| 函数 | 说明 |
|------|------|
| `SBUS_Init()` | 初始化，单字节中断接收启动 |
| `SBUS_Process()` | 喂狗检测，50ms 无数据则失联 |
| `SBUS_GetChannel(ch)` | 获取 1-16 通道原始值 (172-1811) |
| `SBUS_IsValid()` | 返回 1=遥控信号有效，0=失控 |

**协议格式**: 25字节帧，起始 0x0F，结束 0x00/0x04/0x08

### 4.2 舵机控制 (`servo.c`)

| 函数 | 说明 |
|------|------|
| `Servo_Init()` | 初始化 TIM1 PWM，启动 PE9/11/13/14 |
| `Servo_Set180(num, pulse)` | 设置180°舵机角度 (pulse: 500-2500μs) |
| `Servo_Set360(num, speed)` | 设置360°舵机速度 (speed: -1000~+1000) |

**通道映射**:
| 舵机 | 引脚 | 类型 | TIM通道 |
|------|------|------|---------|
| 1号 | PE9 | 360° 驱动 | TIM1 CH1 |
| 2号 | PE11 | 180° 转向 | TIM1 CH2 |
| 3号 | PE13 | 360° 驱动 | TIM1 CH3 |
| 4号 | PE14 | 360° 驱动 | TIM1 CH4 |

**物理补偿值** (freertos.c 全局宏):
```c
#define TRIM_M1  30   // 1号大轮补偿
#define TRIM_M3  30   // 3号小轮补偿
#define TRIM_M4  30   // 4号小轮补偿
```

### 4.3 BMI088 IMU (`BMI088Middleware.c` / `BMI088driver.c`)

| 函数 | 说明 |
|------|------|
| `BMI088_init()` | 初始化，返回 0=成功，非0=错误码 |
| `BMI088_read(gyro, accel, temp)` | 读取传感器数据 |

**数据单位**: gyro [rad/s], accel [m/s²], temp [℃]
**片选**: PA4 (加速度计), PB0 (陀螺仪)
**延时**: `osDelay()` (ms) + `__NOP()` 汇编 (μs)

### 4.4 USB CDC 回传 (`usbd_cdc_if.c`)

- 发送前检查 `hUsbDeviceFS.dev_state == USBD_STATE_CONFIGURED`
- 格式: `G:%d,%d,%d | A:%d,%d,%d\r\n` (数值×1000 取整)
- 例: `G:0,0,1500 | A:0,0,9800\r\n`

## 六、关键配置参数

### 6.1 系统时钟
```
HSE = 8MHz → PLLCLK = 144MHz / 2 = 168MHz (SYSCLK)
APB1 = HCLK / 4 = 42MHz
APB2 = HCLK / 2 = 84MHz
TIM1/TIM8 使用 APB2 (84MHz × 2 = 168MHz)
```

### 6.2 PWM 配置
```
TIM1/TIM8: Prescaler = 167, Period = 19999
PWM频率 = 168MHz / 168 / 20000 = 50Hz
分辩率 = 1μs/pulse
```

### 6.3 SBUS 通道映射
| 通道 | 功能 |
|------|------|
| CH1 | 右手左右 (小轮测试) |
| CH2 | 右手上下 (主驱动/1号轮) |
| CH3 | 左手上下 (转向) |
| CH5 | 副开关 SwE (切换3/4号轮) |
| CH9 | 主开关 SwB (驾驶/调试模式切换) |

## 七、调试方法

1. **编译**: 使用 Keil MDK5，编译器 ARMCC V5.06
2. **串口监控**: 连接 USB 到电脑，打开串口助手 (115200bps) 查看 IMU 数据
3. **遥控器**: 连接 SBUS 信号到 UART3，使用标准 SBUS 接收机
4. **LED 状态**:
   - 蓝灯闪烁 → 系统正常运行
   - 绿灯常亮 → 遥控信号有效
   - 红灯常亮 → 遥控信号丢失（失控保护）

## 八、注意事项

1. **CubeMX 重新生成代码**: `freertos.c` 的 `USER CODE` 区间会被保留，`main.c` 中的任务创建部分可能被覆盖，需重新修改
2. **USB 枚举**: IMU 任务启动后等待 2 秒确保 USB 枚举完成
3. **TRIM 补偿值**: 需根据实际机械装配微调，确保断电时车轮静止
4. **BMI088 延时**: 毫秒级使用 `osDelay()`，微秒级使用 `__NOP()` 汇编（避免 FreeRTOS SysTick 干扰）
