# 三轮遥控车控制代码 - 技术说明文档

## 项目概述

本项目是针对 RoboMaster C板（STM32F407IGHx）开发的三轮遥控车控制程序。通过 SBUS 协议接收遥控器信号，实现对三个 360 度驱动轮和一个 180 度转向舵机的精确控制。

### 主要功能特性

- **双模式控制**：驾驶模式与调试模式通过遥控器 SwB 拨杆切换
- **物理零点补偿 (TRIM)**：解决 360 度舵机硬件公差导致的漂移问题
- **LED 状态指示**：实时显示系统运行状态和信号连接状态
- **失控保护**：信号丢失时自动执行安全保护动作

## 硬件配置

### 主控板
- **型号**: RoboMaster C板
- **MCU**: STM32F407IGHx
- **架构**: ARM Cortex-M4 with FPU

### 舵机配置

| 舵机编号 | 类型 | 用途 | PWM通道 | 接口 |
|---------|------|------|---------|------|
| 1号 | 360度 | 左/右大轮驱动 | TIM1 CH1 | C4 |
| 2号 | 180度 | 转向机构 | TIM1 CH2 | C3 |
| 3号 | 360度 | 左小轮驱动 | TIM1 CH3 | C2 |
| 4号 | 360度 | 右小轮驱动 | TIM1 CH4 | C1 |

### 通信接口
- **SBUS 接收**: USART3 (100000波特率, 9位数据, 偶校验, 2位停止位)
- **调试串口**: USART1 (115200波特率)

### 状态指示灯
- **蓝灯 (PH10)**: 系统心跳，闪烁表示程序正常运行
- **绿灯 (PH12)**: SBUS 信号正常时常亮
- **红灯 (PH11)**: 信号丢失时亮起

### 核心模块说明

#### 1. SBUS 协议解析模块 (sbus.c/sbus.h)

**功能**：解析来自遥控器的 SBUS 协议数据帧

**关键特性**：
- 支持 16 个通道的数据解析
- 兼容多种结束字节 (0x00, 0x04, 0x08)
- 自动检测失控状态和帧丢失

**主要函数**：
```c
void SBUS_Init(void);                      // 初始化SBUS接收
void SBUS_Process(void);                   // 处理超时检测
uint16_t SBUS_GetChannel(uint8_t channel); // 获取通道值 (1-16)
uint8_t SBUS_IsValid(void);                 // 检查数据有效性
```

**通道映射**：
| 通道 | 遥控器输入 | 默认功能 |
|------|-----------|---------|
| CH1 | 右摇杆左右 | 调试模式控制小轮 |
| CH2 | 右摇杆上下 | 主驱动/油门 |
| CH3 | 左摇杆上下 | 转向控制 |
| CH5 | SwE | 选择3号/4号轮 |
| CH9 | SwB | 驾驶/调试模式切换 |

#### 2. 舵机控制模块 (servo.c/servo.h)

**功能**：生成精确的 PWM 信号控制舵机

**PWM 参数**：
- 频率: 50Hz (周期 20ms)
- 分辨率: 1us per count
- 定时器: TIM1 (168MHz 主频, 167分频)

**脉宽定义**：
- 180度舵机: 500μs ~ 2500μs (中位 1500μs)
- 360度舵机: 500μs ~ 2500μs (停止 1500μs, <1500后退, >1500前进)

**主要函数**：
```c
void Servo_Init(void);                              // 初始化所有舵机
void Servo_Set180(uint8_t servo_num, uint16_t pulse_us); // 设置180度舵机角度
void Servo_Set360(uint8_t servo_num, int16_t speed);     // 设置360度舵机速度
void Servo_Stop360(uint8_t servo_num);                   // 停止360度舵机
```

#### 3. 主控制程序 (main.c)

**控制逻辑**：

```
主循环 (10ms 周期)
├── LED 状态更新
├── SBUS 数据处理
├── 模式判断 (SwB 拨杆)
│   ├── 模式1: 驾驶模式
│   │   ├── 右摇杆上下 → 驱动轮前进/后退
│   │   ├── 左摇杆上下 → 转向
│   │   └── 三个轮子按比例协同运动
│   │
│   └── 模式2: 调试模式
│       ├── 右摇杆上下 → 1号大轮
│       ├── 右摇杆左右 + SwE → 3号或4号小轮
│       └── 转向舵机锁定
│
└── 失控保护 (信号丢失时)
    ├── 发送 TRIM 补偿值保持静止
    └── 转向舵机锁定中位
```

## 调参指南

### TRIM 补偿值调整

**问题现象**：摇杆居中时轮子缓慢转动

**调整方法**：
1. 确保遥控器开机，SBUS 信号正常（绿灯亮）
2. 不要触碰任何摇杆
3. 观察哪个轮子在转动
4. 根据以下规则调整 main.c 中的 TRIM 值：
   - 轮子**后退** → 增大 TRIM 值
   - 轮子**前进** → 减小 TRIM 值

**TRIM 参数位置** (main.c 第 126-128 行)：
```c
#define TRIM_M1  30  // 1号大轮补偿值
#define TRIM_M3  30  // 3号小轮补偿值
#define TRIM_M4  30  // 4号小轮补偿值
```

### 死区设置

**当前死区**: ±50 (通道值)

摇杆在中间 ±50 范围内时，系统视为停止输入。这可以避免摇杆轻微漂移导致误动作。

## 失控保护机制

当 SBUS 信号丢失时，系统自动执行以下保护：

1. **驱动轮**：发送 TRIM 补偿值而非 0，确保舵机保持在物理静止位置
2. **转向舵机**：锁定在中位 (1500μs)
3. **LED 指示**：红灯亮起，绿灯熄灭

**注意**：失控时绝对不能发送 0，否则关控后车辆仍会继续运动！

## 故障排查

### SBUS 信号问题

| 症状 | 可能原因 | 解决方案 |
|------|---------|---------|
| 红灯亮 | SBUS 未连接 | 检查 USART3 接口接线 |
| 红灯闪烁 | 信号干扰 | 检查遥控器天线 |
| 遥控器频道错乱 | 通道映射错误 | 检查 SBUS_ParseFrame 函数 |

### 舵机问题

| 症状 | 可能原因 | 解决方案 |
|------|---------|---------|
| 舵机抖动 | PWM 信号不稳定 | 检查电源电压 |
| 轮子空转 | TRIM 值过大 | 减小对应 TRIM 值 |
| 无反应 | 舵机损坏 | 更换舵机 |
